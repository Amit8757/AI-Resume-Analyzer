üìã Project Overview

AI Resume Analyzer is a comprehensive full-stack web application designed to help job seekers optimize their resumes and prepare for interviews. The platform uses intelligent algorithms to analyze resumes against job descriptions, providing ATS (Applicant Tracking System) compatibility scores and actionable feedback.

Key Features

  • ATS Resume Analysis: Upload PDF resumes and analyze them against job descriptions to get compatibility scores (0-100%), keyword matching, and improvement suggestions.
  • Mock Interview Practice: Role-specific interview questions with AI-powered feedback, supporting multiple job categories like Software Engineer, Data Scientist, and Product Manager.
  • Dashboard & Analytics: Comprehensive dashboard showing resume statistics, average ATS scores, and interview performance tracking.
  • User Authentication: Secure JWT-based authentication with user registration, login, and protected routes.

Application Flow

  1. User Authentication: Users register/login ‚Üí JWT token generated ‚Üí Token stored in localStorage
  2. Resume Upload: User uploads PDF ‚Üí Multer processes file ‚Üí PDF-Parse extracts text ‚Üí Stored in MongoDB
  3. ATS Analysis: User provides job description ‚Üí Algorithm analyzes keywords, structure, content ‚Üí Returns score and suggestions
  4. Mock Interview: User selects role ‚Üí Questions generated ‚Üí Answers submitted ‚Üí Feedback provided based on STAR method

ATS Scoring Algorithm

The ATS scorer evaluates resumes using a weighted scoring system:

  • 40% - Keyword Matching: Matches keywords from job description with resume content
  • 30% - Resume Structure: Checks for essential sections (Experience, Education, Skills, etc.)
  • 30% - Content Quality: Evaluates action verbs and quantifiable achievements

‚ùì Important Interview Questions & Answers

Q1: Can you explain your AI Resume Analyzer project?
Answer:
"AI Resume Analyzer is a full-stack MERN application I developed to help job seekers optimize their resumes for ATS systems. The application allows users to upload their resumes in PDF format, analyze them against specific job descriptions, and receive a compatibility score along with actionable feedback.

The platform also includes a mock interview feature where users can practice role-specific questions and receive AI-powered feedback. I implemented secure JWT authentication, file upload handling with Multer, and created a custom ATS scoring algorithm that evaluates resumes based on keyword matching (40%), structure (30%), and content quality (30%)."
Q2: What challenges did you face while building this project?
Answer:
"One major challenge was implementing the PDF parsing functionality. I used the pdf-parse library to extract text from uploaded PDFs, but handling different PDF formats and structures required careful error handling.

Another challenge was designing the ATS scoring algorithm. I researched how real ATS systems work and created a weighted scoring system that analyzes keyword density, resume structure, and content quality. I also had to handle edge cases like missing sections or poorly formatted resumes.

For file uploads, I implemented Multer middleware with proper validation to ensure only PDF files under 5MB are accepted, and I set up proper CORS configuration to handle cross-origin requests between the frontend and backend."
Q3: How did you implement authentication in your application?
Answer:
"I implemented JWT (JSON Web Token) based authentication for secure user sessions. Here's the flow:
  • Registration: User credentials are validated, passwords are hashed using bcrypt with a salt round of 10, and stored in MongoDB
  • Login: User credentials are verified, and a JWT token is generated with a 7-day expiration
  • Protected Routes: I created an authentication middleware that verifies the JWT token from request headers
  • Frontend: The token is stored in localStorage and automatically attached to API requests using Axios interceptors
This ensures that only authenticated users can access features like resume upload, analysis, and mock interviews."
Q4: Explain the database schema for your application.
Answer:
"I designed three main Mongoose schemas:

1. User Schema:
  • name, email (unique), password (hashed)
  • Includes methods for password comparison and JWT token generation
2. Resume Schema:
  • userId (reference to User), originalName, filename, filePath
  • extractedText (from PDF parsing)
  • analysis object containing atsScore, keywordMatches, suggestions, and missingKeywords
  • Timestamps for tracking creation and updates
3. Interview Schema:
  • userId (reference to User), jobRole, status (in-progress/completed)
  • questions array with question text, user answers, and feedback
  • overallScore and completedAt timestamp
All schemas include proper indexing and validation for data integrity."
Q5: How does your ATS scoring algorithm work?
Answer:
"The ATS scoring algorithm uses a weighted approach with three main components:

1. Keyword Matching (40% weight):
  • Extract keywords from the job description
  • Compare with resume content (case-insensitive)
  • Calculate match percentage and identify missing keywords
2. Structure Analysis (30% weight):
  • Check for essential sections: Experience, Education, Skills, Summary
  • Verify proper formatting and organization
3. Content Quality (30% weight):
  • Identify action verbs (achieved, developed, implemented, etc.)
  • Look for quantifiable achievements (numbers, percentages, metrics)
  • Evaluate overall content depth
The final score is calculated as a weighted average and presented on a 0-100 scale with specific suggestions for improvement."
Q6: How did you handle file uploads securely?
Answer:
"I implemented several security measures for file uploads:
  • File Type Validation: Only PDF files are accepted using Multer's fileFilter
  • File Size Limit: Maximum 5MB per file to prevent server overload
  • Unique Filenames: Generated using timestamps and random strings to prevent conflicts
  • Secure Storage: Files are stored outside the public directory with controlled access
  • User Association: Each uploaded file is linked to the authenticated user's ID
  • Error Handling: Comprehensive try-catch blocks with proper error messages
I also implemented cleanup functionality to delete files from the server when users delete resumes from the database."
Q7: What is the difference between authentication and authorization?
Answer:
"Authentication is the process of verifying who a user is (login with credentials), while Authorization is determining what resources a user can access.

In my project:
  • Authentication: Users log in with email/password, and I verify their identity using JWT tokens
  • Authorization: The auth middleware checks if the user has a valid token before allowing access to protected routes. Additionally, users can only view, edit, or delete their own resumes and interviews, not others'
I implemented authorization by checking the userId in the database against the authenticated user's ID from the JWT token."
Q8: How did you manage state in your React application?
Answer:
"I used a combination of React hooks and Context API for state management:
  • useState: For component-level state like form inputs, loading states, and local data
  • useEffect: For side effects like fetching data on component mount and cleanup
  • Context API: Created an AuthContext to manage global authentication state (user info, login/logout functions) accessible across all components
  • Custom Hooks: Created useAuth hook to easily access authentication context
For API calls, I created a service layer (resumeService, interviewService, authService) to separate business logic from components, making the code more maintainable and testable."

‚ö° Quick Technical Questions

Q: What is CORS and why did you need it?
CORS (Cross-Origin Resource Sharing) allows the frontend (localhost:5173) to make requests to the backend (localhost:5000). I configured CORS middleware in Express to accept requests from the client URL.
Q: What is Mongoose and why use it?
Mongoose is an ODM (Object Data Modeling) library for MongoDB. It provides schema validation, type casting, query building, and business logic hooks, making it easier to work with MongoDB in Node.js applications.
Q: Explain the difference between localStorage and sessionStorage.
localStorage persists data even after browser closure, while sessionStorage clears when the tab/window is closed. I used localStorage for JWT tokens to maintain user sessions across browser sessions.
Q: What is the virtual DOM in React?
The virtual DOM is a lightweight copy of the actual DOM. React uses it to efficiently update only the changed parts of the UI by comparing the virtual DOM with the previous version (diffing) and applying minimal updates to the real DOM.
Q: What are React hooks you used?
I used useState for state management, useEffect for side effects and data fetching, useNavigate for programmatic navigation, useContext for accessing global auth state, and created custom hooks like useAuth.
Q: How does bcrypt work?
Bcrypt is a password hashing function that uses a salt (random data) and multiple rounds of hashing to create a secure hash. It's intentionally slow to prevent brute-force attacks. I used bcrypt.hash() to hash passwords and bcrypt.compare() to verify them during login.
